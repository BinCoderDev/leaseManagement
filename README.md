# 公寓管理系统项目文档
## 项目仓库说明
本项目为**前后端分离公寓租赁管理平台**，包含用户移动端、运营后台管理系统；后端基于SpringBoot+MyBatis-Plus，前端Vue3，使用MySQL存储业务数据、Redis做缓存、MinIO管理图片文件，Nginx负责前端静态资源部署。



# 1. 项目概述
## 1.1 项目业务概述
公寓管理系统是公寓租赁平台项目，分为**移动端（租客端）**、**后台管理系统（运营管理端）**：
- 移动端：面向租房用户，提供房源检索、看房预约、租约管理、浏览历史等租房全流程功能；
- 后台管理系统：面向平台管理员，统一管理公寓、房间、租赁订单、租客、后台账号、基础属性配置。


#### 房源检索
用户可按地理位置、租金区间、支付方式等多条件筛选、检索匹配房源，快速定位目标公寓房间。

#### 看房预约管理
用户自主选择空闲时间提交看房预约，线下实地看房，提前和运营人员预约对接。

#### 租约管理
查看个人全部电子租约合同；支持线上发起租约终止、租约延期申请，跟进租约审批状态。

#### 房源浏览历史
系统自动记录用户浏览过的房间/公寓，可随时回看历史房源，辅助租房决策。


#### 公寓信息管理
维护平台所有公寓基础档案，支持新增、编辑、删除公寓基础资料（名称、地址、联系方式等）。

#### 房间信息管理
单公寓下多房间独立管理，维护房间号、户型、面积、租金、配套等房间核心信息，增删改查全操作。

#### 公寓/房间属性管理
统一维护平台通用基础属性字典：公寓配套、房间配套、支付方式、租期、标签、杂费、房间自定义属性等，新增公寓/房间时直接下拉选择，统一规范数据。

#### 看房预约管理
统一查看所有租客提交的看房预约单，修改预约处理状态，安排线下看房接待。

#### 租约管理
管理员创建、编辑、终止租约，生成电子租约合同并推送至租客移动端完成线上签约；管控全生命周期租约状态。

#### 后台系统用户管理
管理运营后台登录账号，新增、编辑、删除、禁用管理员账号，绑定岗位权限。

#### 移动端用户管理
查看全平台租客账号信息，处理租客账户相关问题。

### 1.1.3 核心业务流程
平台核心业务：**签约、续约、退租**

#### 租约7种状态
1. 签约待确认
2. 已签约
3. 已取消
4. 已到期
5. 退租待确认
6. 已退租
7. 续约待确认

## 1.2 项目技术概述
整体采用**前后端分离架构**，各分层技术栈如下：
### 前端
- 核心框架：Vue3

### 后端
- 开发框架：Spring Boot
- ORM持久层：MyBatis、MyBatis-Plus
  > MyBatis-Plus 是MyBatis增强工具，仅增强无侵入，简化CRUD开发、提升开发效率
- Web层：Spring MVC

### 数据存储
1. 关系型数据库：MySQL（存储公寓、房间、租约、用户、基础属性等核心业务数据）
2. 缓存：Redis（缓存字典、首页房源、登录会话、地区数据，减轻DB压力）
3. 对象存储：MinIO
    > 对象存储用于存储图片等非结构化文件，每个文件为独立对象，携带元数据与唯一访问URL。
    > 行业标准：亚马逊S3协议，国内云厂商同类产品：阿里云OSS、华为云OBS、百度云BOS。
    > MinIO为开源兼容S3协议的对象存储，本项目使用MinIO统一存储房源图片、公寓配图。

### 部署
- 前端静态资源服务器：Nginx

## 环境依赖
1. JDK 1.8+
2. MySQL 8.0
3. Redis 6+
4. MinIO 独立对象存储服务
5. Node.js 16+（前端编译）
6. Nginx（静态资源部署）

## 启动顺序
1. 启动MySQL、Redis、MinIO
2. 导入sql初始化库表与基础字典数据
3. 启动后端SpringBoot服务
4. 编译并部署 admin-web / app-web 前端
5. 配置Nginx反向代理，访问对应线上地址
